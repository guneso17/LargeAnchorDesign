Parametric Anchor
Design Platform

A self-initiated engineering tool that replaces manual EN 1992-4:2018 anchor checks with an interactive Python application — seven checks across tension and shear, arbitrary anchor layouts, and live concrete-edge interaction, solved geometrically and reported with one click.

Hilti MO TR Engineering Team
01 The Problem

Checking a multi-anchor base plate against EN 1992-4:2018 means running seven separate checks — steel, pull-out, concrete cone and splitting in tension; steel, pry-out and concrete edge breakout in shear — for every anchor, by hand. Influence areas overlap unpredictably once anchors sit close together or near a free edge, and a spreadsheet has no way to draw that geometry, let alone solve it.

The first version of this logic lived in a MATLAB script. It was later rebuilt from the ground up in Python as an interactive web app — turning a single calculation into a tool any teammate can open in a browser, complete with live geometry, utilization charts and one-click reporting.

02 Engineering Model

Seven checks are implemented end to end — four in tension, three in shear — each reduced to one core geometric question: how much of a theoretical influence area actually lands on real concrete, once anchors overlap each other and the slab edge cuts it off.

  • STEEL — TENSIONTension yield of the anchor rod, As · fuk
  • PULL-OUTCombined bond / concrete failure along the embedment
  • CONCRETE CONETensile breakout, governed by the projected area ratio Acn / A⁰cn
  • SPLITTINGMember-thickness-limited radial cracking near the head
  • STEEL — SHEARShear yield of the anchor rod, V0_Rks = k6 · As · fuk
  • PRY-OUTShear-induced concrete breakout at the rear of the anchor
  • EDGE BREAKOUTDirection-dependent shear breakout near a free edge (X / Y)

That geometric question doesn't have a closed-form answer for an arbitrary anchor layout, so it's solved with computational geometry: every influence zone is built as a polygon, unioned with its neighbors, and clipped against the user-defined slab boundary — for every active failure mode, on every input change.

03 Software Architecture

Built as a single-page Streamlit application wired directly to Hilti's existing PROFIS load outputs. Geometry is computed once with Shapely, then reused for every 2-D and 3-D figure — so the engineering model and what's drawn on screen can never disagree.

State persists across reruns through Streamlit's session state, so a project can be exported to Excel — including an arbitrary number of anchors — and re-imported later without losing a single input.

05 3-D Visualization

The same geometry that drives the calculation is rendered as an interactive 3-D model inside the app — anchor group, base plate and concrete volume, to scale, with the free edge and embedment depth visible at a glance.
